# Discord Sound Player — 開発者向けドキュメント

TRPG セッション中に Discord ボイスチャンネルへ BGM を流すための Electron デスクトップアプリ。

> セットアップ手順・使い方は [USER-GUIDE.md](./USER-GUIDE.md) を参照。

---

## アーキテクチャ

```
Electron App
├── メインプロセス (Node.js)
│   ├── Discord Bot 接続 (discord.js)
│   ├── Voice チャンネル接続・音声送出 (@discordjs/voice)
│   ├── 音声ファイル管理
│   └── IPC ブリッジ（UI ↔ メイン間通信）
└── レンダラープロセス (Web UI)
    ├── 再生コントロール
    ├── ファイル / プレイリスト管理
    └── 設定画面（Bot トークン、接続先チャンネル等）
```

## 機能仕様

### MVP（実装済み）

#### 再生コントロール
- **再生**: 選択したファイルをボイスチャンネルへ流す。デフォルトでループ再生。
- **一時停止 / 再開**: 再生位置を保持したまま停止 / 再開。
- **停止**: 再生を終了し、再生位置をリセット。
- **音量調整**: スライダーで 0〜100% を調整。

#### トラック管理
- OS ネイティブのファイルダイアログで 1 ファイルずつ追加。
- 登録トラックに任意の表示名をつけられる（例: `戦闘BGM`）。
- トラック情報（表示名・ファイルパス）は `userData` 配下の JSON ファイルに永続化。
- 対応フォーマット: `.mp3` / `.ogg` / `.wav` / `.flac`

#### Discord 接続 / 切断
- 複数の Bot トークンを登録し、切り替えて使用できる。
- 接続するサーバー（Guild）とボイスチャンネルをドロップダウンで選択。
- アプリ起動時に前回の接続先を Bot ごとに復元。
- Discord 側から強制退出させられた場合も切断済み状態に自動更新。

### 追加機能（MVP 後に検討）

#### トラック一括登録
- フォルダを指定すると配下の対応ファイルをまとめて登録。

#### プレイリスト
- 複数ファイルをプレイリストに登録・並び替え。
- プレイリスト単位でループ。
- プレイリストの保存・読み込み（JSON）。

#### フェード
- 再生開始時のフェードイン。
- 停止時のフェードアウト。

## 配布形式

| プラットフォーム | 形式 |
|:---|:---|
| Windows | `.exe` インストーラ / ポータブル `.zip` |
| macOS | `.dmg` |
| Linux | `.AppImage` |

electron-builder でビルド・パッケージング。ffmpeg バイナリを同梱する。

GitHub Actions (`discord-sound-build.yml`) が `main` ブランチへの push 時に自動ビルドしてリリースを作成する。
