# Discord Sound Player 仕様書

TRPG セッション中に Discord ボイスチャンネルへ BGM を流すための Electron デスクトップアプリ。

---

## 1. システム構成

### アーキテクチャ

```
Electron App
├── メインプロセス (Node.js)
│   ├── Discord Bot 接続 (discord.js)
│   ├── Voice チャンネル接続・音声送出 (@discordjs/voice)
│   ├── 音声ファイル管理
│   └── IPC ブリッジ（UI ↔ メイン間通信）
└── レンダラープロセス (Web UI)
    ├── 再生コントロール
    ├── ファイル / プレイリスト管理
    └── 設定画面（Bot トークン、接続先チャンネル等）
```

### 起動方法

アイコンをダブルクリック → Electron ウィンドウが開く。

## 3. 機能仕様

### 3-1. MVP（最初に作る機能）

#### 再生コントロール
- **再生**: 選択したファイルをボイスチャンネルへ流す。デフォルトでループ再生。
- **一時停止 / 再開**: 再生位置を保持したまま停止 / 再開。
- **停止**: 再生を終了し、再生位置をリセット。
- **音量調整**: スライダーで 0〜100% を調整。

#### トラック管理
- アプリ内のトラック一覧に音声ファイルを事前登録しておく。
- ファイル追加: OS ネイティブのファイルダイアログで 1 ファイルずつ追加。
- 登録したトラックには任意の表示名をつけられる（例: `戦闘BGM`）。
- トラック情報（表示名・ファイルパス）は `userData` 配下の JSON ファイルに保存。
- 対応フォーマット: `.mp3` / `.ogg` / `.wav` / `.flac`

#### Discord 接続 / 切断
- Bot トークンを設定画面に入力して接続。
- 接続するサーバー（Guild）とボイスチャンネルをドロップダウンで選択。
- アプリ起動時に前回の接続先を復元。
- 「切断」ボタンで Bot をボイスチャンネルから退出させ、再生を停止する。
- Discord 側から強制退出させられた場合（サーバー管理者による切断、チャンネル削除等）も、切断ボタン押下と同様に扱う（再生停止・UI を切断済み状態に更新）。

### 3-2. 追加機能（MVP 後に検討）

#### トラック一括登録
- フォルダを指定すると、配下の対応フォーマットのファイルをまとめて登録。
- 表示名はファイル名から自動設定（後から個別に変更可能）。

#### プレイリスト
- 複数ファイルをプレイリストに登録・並び替え。
- プレイリスト単位でループ（全曲ループ）。
- プレイリストの保存・読み込み（JSON ファイル）。

#### フェード
- 再生開始時のフェードイン。
- 停止時のフェードアウト。

## 5. 設定項目

| 項目 | 説明 |
|---|---|
| Bot トークン | Discord Developer Portal で発行したトークン |
| デフォルト音量 | 起動時の初期音量（0〜100）|
| 前回の接続先を復元 | 起動時に前回の Guild / Channel へ自動接続するか |

設定は `userData` 配下の JSON ファイルに保存（平文）。

## 6. 使用フロー

### セッション前（準備）

1. アプリを起動し、設定画面で Bot トークンを入力。
2. トラック一覧に BGM ファイルを登録・表示名をつけておく。

### セッション開始時

1. アプリを起動（前回の接続先が自動復元される）。
2. GM と参加者が Discord のボイスチャンネルに入る。
3. アプリの接続先でサーバーとチャンネルを選択し「接続」ボタンを押す。
4. Bot がボイスチャンネルに参加する（参加者には Bot が入室したように見える）。

### セッション中

- GM はローカルのアプリ画面からトラックを選んで再生・停止・音量調整。
- **Discord 側での操作は不要**。参加者はボイスチャンネルで音楽が流れてくるのを聞くだけ。

### セッション終了時

1. アプリの「停止」ボタンで再生を終了。
2. 「切断」ボタンで Bot をボイスチャンネルから退出させる。
3. アプリを閉じる。

---

## 7. Discord Bot の準備（ユーザー向け）

1. [Discord Developer Portal](https://discord.com/developers/applications) でアプリを作成。
2. Bot を作成してトークンを取得。
3. OAuth2 で以下のスコープを付与してサーバーへ招待:
   - `bot`
   - 権限: `Connect` / `Speak`
4. アプリの設定画面にトークンを入力。
