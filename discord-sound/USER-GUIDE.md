# Discord Sound Player — 使い方ガイド

TRPG セッション中に Discord ボイスチャンネルへ BGM を流すためのデスクトップアプリ。GM がローカルで操作するだけで、ボイスチャンネルの参加者全員に音楽が届く。

---

## インストール

[Releases](https://github.com/nagashima-w/trpg-tool/releases) から最新版をダウンロードしてインストールする。

| OS | ファイル |
|:---|:---|
| Windows | `.exe` インストーラ（または `.zip` ポータブル版） |
| macOS | `.dmg` |
| Linux | `.AppImage` |

---

## Discord Bot の準備

アプリを使うには、音楽を流す専用の Discord Bot が必要。

1. [Discord Developer Portal](https://discord.com/developers/applications) でアプリを作成
2. **「Bot」** タブで **「Add Bot」→「Reset Token」** してトークンを取得・保存
3. **「OAuth2 → URL Generator」** で以下を設定してサーバーへ招待:
   - Scopes: `bot`
   - Bot Permissions: `Connect` / `Speak`

> Bot トークンは他人に見せないこと。見せてしまった場合は即「Reset Token」で無効化する。

---

## 初回セットアップ

1. アプリを起動すると設定画面が自動で開く
2. **「Bot を追加」** 欄に **名前**（例: `セッションBot`）と **Bot トークン** を入力して「追加」
3. 追加した Bot がアクティブになり、参加しているサーバー一覧が読み込まれる
4. 接続するサーバーとボイスチャンネルを選択して **「接続」**

---

## BGM トラックの登録

1. トラック一覧の **「+ 追加」** ボタンをクリック
2. OS のファイルダイアログで音声ファイルを選択（対応形式: `.mp3` / `.ogg` / `.wav` / `.flac`）
3. 表示名を任意で設定（例: `戦闘BGM`、`街の雑踏`）

登録したトラックはアプリを閉じても保持される。

---

## セッション中の使い方

### セッション開始時

1. アプリを起動（前回の接続先が自動復元される）
2. GM と参加者が Discord ボイスチャンネルに入る
3. アプリでサーバーとチャンネルを確認して **「接続」**
4. Bot がボイスチャンネルに参加する

### 再生操作

トラック一覧からファイルを選んで **▶** を押すだけ。

| ボタン | 動作 |
|:---|:---|
| ▶ | 再生（デフォルトはループ） |
| ⏸ | 一時停止（再生位置を保持） |
| ⏹ | 停止（再生位置をリセット） |
| 🔁 | ループ ON / OFF |

音量スライダーで 0〜100% を調整できる。Discord 側での操作は不要。

### セッション終了時

1. **「停止」** で再生を終了
2. **「切断」** で Bot をボイスチャンネルから退出
3. アプリを閉じる

---

## 複数 Bot の登録・切り替え

Discord サーバーごとに専用の Bot を使い分けたい場合など、複数の Bot を登録できる。

1. **⚙ 設定** を開く
2. **「Bot を追加」** 欄で追加の Bot の名前とトークンを入力して「追加」
3. Bot 一覧のラジオボタンで切り替えると即座に再ログインされる
4. 各 Bot は最後に接続したサーバー・チャンネルを個別に記憶する

> トークンは末尾 4 文字のみ表示される。

---

## 設定項目

| 項目 | 説明 |
|:---|:---|
| Bot トークン | Discord Developer Portal で発行したトークン |
| デフォルト音量 | 起動時の初期音量（0〜100） |
| 前回の接続先を復元 | 起動時に前回の Guild / Channel へ自動接続するか |

設定は OS の `userData` フォルダ内の JSON ファイルに保存される。
